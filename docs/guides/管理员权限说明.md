## 商家审核流程

### 1. 用户提交申请
- 用户在"个人中心" → "商家服务" → "立即申请"
- 填写4步申请表单：
  - 基本信息（姓名、电话、身份证号、身份证照片）
  - 店铺信息（名称、类型、简介、地址、营业时间、Logo）
  - 资质证明（营业执照、统一社会信用代码、法人姓名、其他资质）
  - 确认提交

### 2. 管理员审核
- 管理员登录后台
- 进入"商家管理"页面
- 查看"待审核"列表
- 点击"审核申请"按钮
- 查看商家详细信息
- 选择审核结果（通过/拒绝）
- 填写审核意见（拒绝时必填）
- 提交审核

### 3. 审核结果
- **通过**: 用户角色自动变为"商家"，可以进入商家中心
- **拒绝**: 用户收到拒绝通知，可以重新申请

## 管理员操作指南

### 登录系统
1. 访问 http://localhost:3000/login
2. 输入账号: zh1111
3. 输入密码: SLG123
4. 点击登录

### 进入商家管理
1. 登录后，在导航栏找到"管理员中心"
2. 点击"商家管理"
3. 进入商家审核页面

### 审核商家申请
1. 在"待审核"标签页查看所有待审核申请
2. 点击某个商家的"审核申请"按钮
3. 查看商家详细信息：
   - 基本信息（姓名、电话、身份证号）
   - 店铺信息（名称、类型、地址、营业时间）
   - 资质信息（营业执照、信用代码、法人）
   - 资质文件（身份证照片、营业执照、其他资质）
4. 选择审核结果：
   - ✅ 通过：用户成为商家，可以进入商家中心
   - ❌ 拒绝：需要填写拒绝原因
5. 点击"提交审核"

### 查看商家详情
1. 在商家列表中点击"查看详情"
2. 查看完整的商家信息
3. 查看审核记录（如果已审核）

### 管理商家店铺
1. 对于已通过的商家，点击"管理店铺"
2. 进入商家中心，查看和管理该商家的商品和订单

## 注意事项

1. **审核时效**: 建议在1-3个工作日内完成审核
2. **审核标准**: 
   - 身份信息真实有效
   - 营业执照齐全
   - 店铺信息完整
   - 符合平台入驻规范
3. **拒绝原因**: 拒绝申请时必须填写详细原因，方便用户改进
4. **数据安全**: 管理员操作会记录日志，便于追溯
5. **权限控制**: 不同角色有不同的权限，请按照职责范围操作

## 技术实现

### 前端路由守卫
```javascript
// 需要管理员权限的页面
meta: { requiresAdmin: true }

// 权限验证
if (requiresAdmin) {
  if (userRole !== '管理员') {
    next('/') // 无权限，跳转首页
  }
}
```

### 后端API接口
```
GET  /api/v1/admin/merchants - 获取商家列表
POST /api/v1/admin/merchants/:id/audit - 审核商家申请
GET  /api/v1/admin/merchants/:id - 获取商家详情
```

### 数据库表结构
- t_merchant_applications - 商家申请表
- t_users - 用户表（包含角色字段）
- t_merchants - 商家信息表
- t_audit_logs - 审核日志表
